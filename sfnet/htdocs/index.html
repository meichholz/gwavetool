logools - Tools for monitoring volatile logfiles

Please refer to the main project page for releases, news and stuff.

Feel free to browse the CVS Tree or see checkout instructions.

Motivation and Goal

The logtools provide functionality missing in - say - tail and tee. The basic idea is to pipe the new lines in a logfile through analysing processing in real time.

There are some issues with tail -f and tee:

  • When the system ist rebooted, the log analysis must continue from the last point. A status must be maintained.
  • When the log file is cycled and renamed, it does not grow any more. Thus, it must be reopened.
  • When the log file is truncated ("zeroed"), there must be "rewinding".
  • The output must be distributed to processes, not files.

As an answer, there are three tools:

  • tailfd: the tail -f daemon
  • teepee: the tee for processes
  • tailfdx : an extended daemon providing both functionality with a configuration file.

The tools are in use in our mailcluster (analysing some Million log entries per day).
